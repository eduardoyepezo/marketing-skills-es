# Ideas de Experimentos de Onboarding

Lista exhaustiva de pruebas A/B y experimentos para el onboarding y la activación de usuarios.

## Contenidos
- Experimentos de Simplificación del Flujo (reducir fricción, secuenciación de pasos, progreso y motivación)
- Experimentos de Experiencia Guiada (tours del producto, optimización de CTA, orientación de UI)
- Experimentos de Personalización (segmentación de usuarios, contenido dinámico)
- Experimentos de Victorias Rápidas y Engagement (tiempo hasta el valor, mecánicas de motivación, soporte y ayuda)
- Experimentos de Email y Multi-Canal (emails de onboarding, contenido de email, bucles de feedback)
- Experimentos de Re-engagement (recuperación de usuarios parados, experiencia de retorno)
- Experimentos Técnicos y de UX (rendimiento, onboarding móvil, accesibilidad)
- Métricas a Rastrear

## Experimentos de Simplificación del Flujo

### Reducir Fricción

| Prueba | Hipótesis |
|------|------------|
| Momento de verificación de email | Durante vs. después del onboarding |
| Estados vacíos vs. datos de ejemplo | Ejemplos pre-rellenados |
| Plantillas pre-rellenadas | Acelerar la configuración con plantillas |
| Opciones de OAuth | Vinculación de cuenta más rápida |
| Cantidad de pasos requeridos | Menos pasos requeridos |
| Campos opcionales vs. requeridos | Minimizar requisitos |
| Opciones de omitir | Permitir saltarse pasos no críticos |

### Secuenciación de Pasos

| Prueba | Hipótesis |
|------|------------|
| Orden de los pasos | Probar diferentes secuencias |
| Ordenamiento de valor primero | Funciones de mayor valor primero |
| Colocación de fricción | Mover pasos difíciles más tarde |
| Balance requerido vs. opcional | Proporción de pasos requeridos |
| Caminos únicos vs. ramificados | Un camino vs. personalizado |
| Inicio rápido vs. configuración completa | Camino mínimo al valor |

### Progreso y Motivación

| Prueba | Hipótesis |
|------|------------|
| Barras de progreso | Mostrar porcentaje de finalización |
| Longitud de la lista de verificación | 3-5 ítems vs. 5-7 ítems |
| Gamificación | Insignias, recompensas, logros |
| Mensajes de finalización | Visibilidad de "X% completado" |
| Punto de partida | Comenzar al 20% vs. 0% |
| Momentos de celebración | Reconocer las finalizaciones |

## Experimentos de Experiencia Guiada

### Tours del Producto

| Prueba | Hipótesis |
|------|------------|
| Tours interactivos | Herramientas como Navattic, Storylane |
| Orientación por tooltip vs. modal | Sutil vs. llamativo |
| Tutoriales en video | Para flujos de trabajo complejos |
| Autodirigido vs. guiado | Control del usuario vs. estructurado |
| Longitud del tour | Más corto vs. exhaustivo |
| Disparador del tour | Automático vs. iniciado por el usuario |

### Optimización de CTA

| Prueba | Hipótesis |
|------|------------|
| Variaciones de texto de CTA | Prueba de copy orientado a la acción |
| Colocación del CTA | Posición dentro de las pantallas |
| Tooltips en la app | Prompts de descubrimiento de funciones |
| CTAs pegajosos | Persistir durante el onboarding |
| Contraste del CTA | Prominencia visual |
| CTAs secundarios | "Saber más" vs. solo primario |

### Orientación de UI

| Prueba | Hipótesis |
|------|------------|
| Highlights de hotspot | Llamar la atención sobre funciones clave |
| Coachmarks | Consejos contextuales |
| Anuncios de funciones | Descubrimiento de nuevas funciones |
| Ayuda contextual | Ayuda donde los usuarios la necesitan |
| Búsqueda vs. guiado | Autoservicio vs. dirigido |

## Experimentos de Personalización

### Segmentación de Usuarios

| Prueba | Hipótesis |
|------|------------|
| Onboarding basado en rol | Diferentes caminos por rol |
| Caminos basados en objetivos | Personalizar por objetivo declarado |
| Dashboards específicos por rol | Vistas predeterminadas relevantes |
| Pregunta de caso de uso | Personalizar basado en la respuesta |
| Caminos específicos por industria | Personalización vertical |
| Basado en experiencia | Caminos para principiantes vs. expertos |

### Contenido Dinámico

| Prueba | Hipótesis |
|------|------------|
| Bienvenida personalizada | Nombre, empresa, rol |
| Ejemplos de la industria | Casos de uso relevantes |
| Recomendaciones dinámicas | Basadas en respuestas del usuario |
| Sugerencias de plantillas | Pre-rellenadas para el segmento |
| Destacado de funciones | Relevantes para los objetivos declarados |
| Datos de benchmark | Métricas específicas de la industria |

## Experimentos de Victorias Rápidas y Engagement

### Tiempo hasta el Valor

| Prueba | Hipótesis |
|------|------------|
| Primera victoria rápida | "Completa tu primer X" |
| Mensajes de éxito | Después de acciones clave |
| Celebraciones de progreso | Momentos de hito |
| Sugerencias del siguiente paso | Después de cada finalización |
| Demostración de valor | Mostrar lo que lograron |
| Vista previa del resultado | Cómo se ve el éxito |

### Mecánicas de Motivación

| Prueba | Hipótesis |
|------|------------|
| Insignias de logro | Elementos de gamificación |
| Rachas | Engagement de días consecutivos |
| Tablas de clasificación | Comparación social (si es apropiado) |
| Recompensas | Incentivos por finalización |
| Mecánicas de desbloqueo | Funciones reveladas progresivamente |

### Soporte y Ayuda

| Prueba | Hipótesis |
|------|------------|
| Llamadas de onboarding gratuitas | Para productos complejos |
| Ayuda contextual | A lo largo del onboarding |
| Soporte de chat | Disponibilidad durante el onboarding |
| Alcance proactivo | Para usuarios atascados |
| Recursos de autoservicio | Documentos de ayuda, videos |
| Acceso a la comunidad | Soporte de pares tempranamente |

## Experimentos de Email y Multi-Canal

### Emails de Onboarding

| Prueba | Hipótesis |
|------|------------|
| Email de bienvenida del fundador | Personal vs. genérico |
| Disparadores basados en comportamiento | Basado en acción/inacción |
| Momento del email | Inmediato vs. retrasado |
| Frecuencia de email | Más vs. menos contactos |
| Formato de consejos rápidos | Contenido corto y accionable |
| Video en el email | Formato más atractivo |

### Contenido de Email

| Prueba | Hipótesis |
|------|------------|
| Líneas de asunto | Optimización de tasa de apertura |
| Profundidad de personalización | Nombre vs. basado en comportamiento |
| Prominencia del CTA | Acción única y clara |
| Inclusión de prueba social | Testimonios en el email |
| Mensajes de urgencia | Recordatorios de prueba |
| Texto plano vs. diseñado | Prueba de formato |

### Bucles de Feedback

| Prueba | Hipótesis |
|------|------------|
| NPS durante el onboarding | Cuándo preguntar |
| Pregunta de bloqueo | "¿Qué te impide avanzar?" |
| Seguimiento de NPS | Acciones basadas en la puntuación |
| Feedback en la app | Pulgares arriba/abajo en funciones |
| Momento de la encuesta | Cuándo solicitar feedback |
| Incentivos de feedback | Recompensar por completar |

## Experimentos de Re-engagement

### Recuperación de Usuarios Parados

| Prueba | Hipótesis |
|------|------------|
| Momento del email de re-engagement | Cuándo enviar |
| Alcance personal | Humano vs. automatizado |
| Camino simplificado | Pasos reducidos para los que regresan |
| Ofertas de incentivo | Descuento o prueba extendida |
| Identificación del problema | Preguntar qué está bloqueando |

### Experiencia de Retorno

| Prueba | Hipótesis |
|------|------------|
| Mensaje de bienvenida de vuelta | Reconocer el retorno |

| Estado cambiado | Qué pasó mientras estuvo fuera |
| Re-onboarding | Opción de comenzar de nuevo |
| Mensajes de urgencia | Tiempo de prueba restante |

## Experimentos Técnicos y de UX

### Rendimiento

| Prueba | Hipótesis |
|------|------------|
| Optimización del tiempo de carga | Más rápido = mayor finalización |
| Carga progresiva | Rendimiento percibido |
| Capacidad sin conexión | Experiencia móvil |
| Manejo de errores | Recuperación elegante de fallos |

### Onboarding Móvil

| Prueba | Hipótesis |
|------|------------|
| Objetivos táctiles | Tamaño y espaciado |
| Navegación por deslizamiento | Patrones nativos de móvil |
| Cantidad de pantallas | Menos pantallas necesarias |
| Optimización de input | Formularios amigables para móvil |
| Momento del permiso | Cuándo preguntar |

### Accesibilidad

| Prueba | Hipótesis |
|------|------------|
| Soporte de lector de pantalla | Impacto de la accesibilidad |
| Navegación por teclado | Usuarios sin ratón |
| Contraste de color | Visibilidad |
| Tamaño de fuente | Legibilidad |

## Métricas a Rastrear

Para todos los experimentos, medir:

| Métrica | Descripción |
|--------|-------------|
| Tasa de activación | % que alcanza el evento de activación |
| Tiempo hasta la activación | Horas/días hasta el primer valor |
| Tasa de finalización de pasos | % que completa cada paso |
| Puntos de abandono | Dónde los usuarios abandonan |
| Tasa de retorno | Usuarios que regresan |
| Retención día 1/7/30 | Engagement a lo largo del tiempo |
| Adopción de funciones | Qué funciones se usan |
| Solicitudes de soporte | Volumen durante el onboarding |
